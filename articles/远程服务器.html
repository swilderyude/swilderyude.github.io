Tmux

• 防止实验中断 下载

tmux -V # 查看版本

#### Ubuntu / Debian 系统（最常用）
apt update             # 先更新软件列表
apt install tmux       # 执行安装
### 创建会话
tmux new -s seesion_name
### 连接会话
tmux at -t seesion_name
### 退出会话
tmux detach
### 查看当前会话
tmux ls

### 快捷键
1. 进入 tmux 后。 
2. 按下 Ctrl + B，然后按 %（左右分屏）。
3. 按下 Ctrl + B，然后按 "（上下分屏）。
4. 按下 Ctrl + B，然后按 方向键 可以在不同格子里切换。

先输入上述对话，再激活环境进行训练。 和他人共用的话，tmux会被误清理。 配合nohop训练

nohup python train.py > train.log 2>&1 &

使用 nohup ... & 将程序放到后台运行后，所有的输出（原本在屏幕上跳动的进度条、日志、报错等）都会被写入你指定的 train.log 文件中。

  1. 实时查看进度 让屏幕像之前训练时一样实时滚动显示进度，可以使用 tail 命令：
tail -f train.log

  • -f：代表 follow，它会盯住这个文件，只要有新内容（比如下一轮训练开始了），它就会立刻显示在屏幕上。

  • 如何退出？ 按下 Ctrl + C。

  • 注意：在 tail -f 模式下按 Ctrl + C 只会停止查看日志，你的训练任务依然在后台继续运行，不会受到任何影响。

  1. 一次性查看全部内容 只想看看现在跑到第几轮了，或者看看有没有报错，可以使用：
# 查看文件最后 50 行
tail -n 50 train.log

# 或者查看整个文件（适合文件还不是很大的时候）
cat train.log
  1. YOLOv8 特有的结果查看 YOLOv8 除了把文本记在 train.log 里，它还会自动在你的项目目录下创建一个文件夹（比如 runs/train/yolov8n-spot/）：

  • results.csv：这里记录了每一轮的损失（Loss）和准确率（mAP）。

  • args.yaml：记录了你训练时用的所有参数。

  • last.pt / best.pt：正在生成的模型权重。 你可以通过 ls 命令查看这些文件是否在不断更新：

ls -lh runs/train/yolov8n-spot/

  • 新建：screen -S train  

  • 退出（分离）：Ctrl + A 然后按 D

  • 找回：screen -r train 退出当前 screen 会话

  • 按下 Ctrl + A，然后按 d，这会让 screen 会话进入“分离”状态。分离后，screen 会话会在后台继续运行，你可以稍后重新附加（reattach）到该会话。 结束一个 screen 会话

  • 输入 exit 并按回车键 screen -X -S [会话ID] quit 对于鼠标滚动问题： 打开配置文件

nano ~/.screenrc

在末尾添加文本

termcapinfo xterm* ti@:te@

Ctrl+O 回车保存，Ctrl+X 退出
