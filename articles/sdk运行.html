  SDK 方案（v1）：

  - 头文件用 C ABI (extern “C”) → C/C++/# 都能直接对接（C# 走 P/Invoke）

  - 实现层用 C++ 写薄封装；推理直接复用现有 app/src/pipeline.py，通过启动 Python

   子进程驱动（保证离线 + 不重写已经验证过的逻辑）

  - C++ Demo 作为可运行示例，需求里只要求一种语言的 Demo

  - JSON/CSV 输出 由参数切换，异常也写空文件（按需求）

3.对比app

结论：SDK 不等同于 app 的完整功能版。它更像一个“单图检测 C ABI 对接层”，如果目标是满足外部系统调用“本地图片路径 -> JSON/CSV 结果”的最小 SDK，基本成型；但如果要求覆盖 app 已实现的功能，当前 SDK 明显不完整。

主要差异：

功能 | app | SDK 当前情况 |

|—|—|—|

单图检测 | 有 | 有，spot_detect_image() |
文件夹批量检测 | 有 | 没有对外 API |
单类/三类模式和类别别名 | 有 | 只传模型路径，不传 app 的 alias/命名规则 |
滑窗参数 | 有 | 有：tile_size / overlap / use_sliding_window |
Face mask 详细参数 | 有：最小面积、平滑核、最大连通域、内部 scale | SDK 只暴露 use_face_mask，细节没暴露 |
可视化图 | app 输出并展示 | pipeline 会生成，但 SDK 只返回 JSON/CSV 路径，不稳定暴露 |
裁剪图和 crops.zip | app 输出并可导出 | pipeline 会生成到 _runs，SDK API 不返回 |
CSV 字段 | app 有 12 列：含 crop_path/subtype/subtype_score | SDK CSV 只写 9 列，丢了这 3 个字段 |
JSON 字段 | app 完整 | SDK JSON 基本保留完整 rows |
BMP/WEBP | app 支持 | SDK 只允许 JPG/JPEG/PNG |
UI 预览、翻页、缩放、日志、另存 | 有 | SDK 不包含，合理 |

关键代码位置：

  • SDK 对外 API 只有单图检测：SDK/include/spot_detect.h

  • SDK runner 确实复用了 app 的 YOLOSpotDemo.run()：SDK/src/runner/run_detect.py

  • app 的完整单图 pipeline 会产出可视化、CSV、JSON、crops zip：app/src/pipeline.py

  • app 的文件夹批量检测和 summary/all_detections 输出在这里：app/desktop_app.py

还有一个重要判断：SDK 不是完全独立包。它的 runner 通过相对路径 import ../app/src/pipeline.py，所以离开当前项目结构后不能单独工作，除非把 app 依赖、模型、assets、runner 打包进去。

所以我会这样定性：作为 v1 最小交付 SDK：基本可用；作为“相较 app 功能完整”的 SDK：不完整。 最优先补的是批量接口、完整 CSV 字段、返回可视化/crops 路径、Face mask 详细参数，以及把 runner/app 依赖打成可独立分发包。

4.补充功能

已经把 SDK 改成可独立交付版了。现在对方只拿整个 SDK 文件夹，不需要同级 app/ 目录也能跑。

主要改动：

  • 把 app 的推理代码复制进 SDK/python/src。

  • 把模型复制进 SDK/models。

  • 把 facexlib 离线权重复制进 SDK/assets/facexlib/weights，包含 RetinaFace 和 parsing 权重，避免离线机器运行时下载。

  • 修改 run_detect.py，现在只从 SDK 自己目录推导路径，不再 import ../app/src/pipeline.py。

  • 修改 pipeline.py，face parsing 权重固定从 SDK 内置 assets 加载。

  • 新增 requirements.txt 和 MANIFEST.md。

  • 更新 README.md 和 smoke_test.sh，默认模型路径改为 SDK 内部。

  • CSV 输出补齐为 app 同款 12 列：crop_path/subtype/subtype_score 不再丢。

  • 补了 runner 内部超时保护，避免 Python 长时间无 stdout 时 C++ popen 超时失效。

验证过：

  • cmake --build SDK/build --config Release 通过。

  • python3 -m py_compile 通过。

  • 把 SDK 复制到 /tmp/SpotSDKStandalone/SDK 后，在没有 app/ 的独立目录里直接跑 runner 成功。

  • 短超时测试成功：1 秒超时会退出并生成空合规 CSV。

现在 SDK 大约 170M，主要是模型和 facexlib 离线权重。交付时不要只给动态库和头文件，要给整个 SDK/ 文件夹。
