深度可分离卷积

基于深度可分离卷积来对卷积层进行优化和改进，从而降低参数量，减少计算量，加速运算，使网络更加轻量化。

图片未发布：Pasted image 20260119202810.png 一般卷积是一个多通道的卷积核在一个多通道的输入里面滑动，把每次滑动位置的卷积核的权重和原始输入的对应像素相乘求和，把这个相乘求和的值，填在右边生成的feature map上面，那输入有几个通道，卷积核有几个通道，比如第一层RGB三个通道，第一层的卷积核也应该有三个通道来对应RGB3个输入通道。 滑动的区域是感受也，把感受也里的像素和卷积核的像素对应位置相乘求和，把数量积的结果填在右边，一个卷积核生成一个feature map
图片未发布：Pasted image 20260120130950.png 卷积核都是三通道，输入也是三通道。
图片未发布：Pasted image 20260120133840.png 卷积核在里面滑动，在输入的tensor里滑动，每滑动一次就是卷积一次，每一次卷积是卷积核的权重和原始输入的像素和对应位置相乘求和。 生成的feature作为下一次网络的输入，仍然是相乘求和。
图片未发布：Pasted image 20260120140641.png 深度可分离卷积，输入三个通道，分别用三个卷积核对这三个通道进行处理，输出三个feature map叠起来输入下一层。
图片未发布：Pasted image 20260120140929.png 下一步处理就是用1X1卷积来对跨通道的信息来处理。1X1卷积的作用就是跨层的信息融合。
图片未发布：Pasted image 20260120141228.png 先深度可分离depth wise卷积，再1X1卷积，即point wise卷积，或者叫点卷积，逐点卷积。 depth wise处理的是长宽方向的空间信息，他不关心跨通道信息，而point wise只关心跨通道信息，它不关心床宽方向的信息。
图片未发布：Pasted image 20260120141614.png
图片未发布：Pasted image 20260120144355.png 先depth wise，每一个通道用自己通道卷积核处理，在用point wise卷积来对跨通道的信息处理，就是整个的基本模块。

图片未发布：Pasted image 20260120154826.png 深度可分离卷积是分组卷积的特例。

图片未发布：Pasted image 20260120161012.png Group Convolution是一个卷积核不再处理所有的通道，而是只处理一部分通道，以往是一个卷积核处理所有通道。depthwise可以看作是Group的极速。 分组卷积在论文里是Cardinality参数，分组卷积的组数，可以大大提升性能。

图片未发布：Pasted image 20260121150927.png 分离卷积分为空间可分离卷积和深度可分离卷积。空间可分离卷积就是非对称卷积，但其轻量化效果不如深度可分离卷积。

MobileNet V1计算性能分析

为了其易于部署，让计算机能高效运行，所以分析计算性能。

图片未发布：Pasted image 20260121182924.png 为什么1X1卷积比较好加速？
图片未发布：Pasted image 20260121183138.png 卷积的过程就是粉色和黄色向量点乘，逐元素相乘求和，将结果填在红色区。 1X1卷积作用：能够起到跨层通信，引入额外的非线性进行降维或升维。 卷积在代码要循环，我们希望是是并行的。 高效原因： 引入im2col，把卷积运算变成矩阵乘法计算。
图片未发布：Pasted image 20260121183621.png 每一次卷积的输入变为列向量，四次卷积的输入变为四列的矩阵。
图片未发布：Pasted image 20260121191613.png 卷积核为行向量，与矩阵每一列相乘，做数量积，实现把卷积运算变为矩阵运算。 矩阵运算借助GEMM库进行高度的优化和加速
图片未发布：Pasted image 20260121192122.png 传统卷积需要im2col，但1X1卷积天生就是一个向量，不需要经过这一步，因此能够更高效的计算。 MobileNet里1X1卷积占了90%的计算量，所以这个网络很快。

MobileNet V3

V3包括MobileNetV3-Large和MobileNetV3-Small两个版本，以满足不同的计算需求。按照论文给出的实验结果，V3-Large比V2的ImageNet准确率提高3.2%，速度快20%。在COCO数据集上，V3-Large与V2的准确率相当，但是速度快25%。

图片未发布：Pasted image 20260122123841.png 蓝色代表v3large宽度系数调整版（1），宽度系数越大，意味着模型的参数越多，计算量越大，准确率越来越高。 紫色虚线对应分辨率调整版（2） 重点看v3与v2的不同 latency在手机上推理所需要的时间

改进

为了取得准确率和推理速度的完美平衡，V3模型主要做了以下四方面技术改进。 （1）采用增加了SE机制的Bottleneck（反向线性残差）模块结构。 在V2采用的反向线性残差块基础上增加SE模块，提升模块学习效率。V3版反向线性残差块的结构如图2.20所示。NL表示非线性激活函数。

图片未发布：Pasted image 20260122131838.png SE模块是对特征层采取全局平均池化，第一个FC全连接，第二个全连接（H-睡个毛函数），再乘以这个主分支往下传过来的特征层，得到新的输出。 多了一个SE模块进行变换。
图片未发布：Pasted image 20260122132659.png SE主分支往下走，另一个分支，再相乘。 用例子演示SE逻辑。
图片未发布：Pasted image 20260122132920.png （2）采用h-swish 替代MobileNetV2 中的ReLU6 激活函数，提升模型推理速度。
图片未发布：Pasted image 20260122133212.png 图2.22对比了 h-sigmoid 与 sigmoid 的函数曲线，表明h-sigmoid 可以取得与 sigmoid相当的效果。
图片未发布：Pasted image 20260122133249.png 直线状态机算量大大降低。
图片未发布：Pasted image 20260122133431.png 推理速度加快。 （3）采用更为精细的 NAS 搜索方法建立基准模型。 论文中介绍首先采用 NAS全局搜索确定模块的最优结构，论文中称这个操作为Platform-Aware NAS for Blockwise Search。然后做模块内的分层搜索确定最优卷积核数量，论文中称力 NetAdapt for Layerwise Search。 v2中卷积核数量都是3X3的，v3不同 （4）进一步优化模型的输入 Stage 与输出 Stage 的结构，在不降低准确率的前提下，降低计算量，提升速度。 MobileNetV2 的第一个卷积层采用32个过滤器，MobileNetV3減少为16个，降低计算量的同时仍然能够保持准确率。 对于最后一个 Stage 的优化方案如图2.24所示。对比基于 NAS 得到的原始结构，在不损失模型准确率的前提下，优化后的结构舍弃了其中比较耗费计算力的三个层。

v1-3总结

MobileNetV1有两个创新点： （1） 采用深度可分离卷积（Depthwise Separable Convolution）取代标准卷积（Standard Convolution），降低计算量。 （2）采用a、p两个超参数，调节模型的宽度（卷积核个数）和图像分辦率，控制模型规模，满足不同应用需求。 正如论文题目所声称的那样，MobileNetV2其主要技术改进有两点： （1） 采用反向残差结构 （Inverted residuals） （2） 采用线性瓶颈层 （Linear Bottlenecks） 即反向线性残差结构。 为了取得准确率和推理速度的完美平衡，V3与V2相比，模型主要做了以下四方面技术改进。 （1）采用增加了SE机制的Bottleneck模块结构。 （2）采用h-swish替代MobileNetV2中的ReLU6激活函数，提升模型推理速度。 （3）采用更为精细的NAS搜索方法建立基准模型。 （4）进一步优化模型的输入Stage与输出Stage的结构，在不降低准确率的前提下，降低计算量，提升速度。
